# 要件定義書：家庭向け食材在庫管理・レシピ提案Webアプリ

## 1. プロジェクト概要

### 目的
家庭の食材在庫を管理し、現在ある食材をもとに作れる定番レシピを提案するWebアプリケーションを開発する。「今日の夕飯何にしよう」という日常的な課題を、在庫情報と連動したレシピ提案によって解決する。

### 対象ユーザー
実家の家族（家族全員で在庫を共有するクローズドな利用を想定）

---

## 2. 機能要件

### 2.1 認証機能
- メールアドレスとパスワードによるユーザー登録・ログイン
- ログアウト機能
- 在庫は家族全員で共有する（アカウントに紐づかず、世帯単位で管理）

### 2.2 食材在庫管理機能
- 食材はプルダウン形式で選択（自由入力なし）
- 食材ごとに量（数値）を登録・更新・削除できる
- 在庫一覧を表示できる

### 2.3 レシピ提案機能
- レシピ提案画面で「今日の人数」を入力できる（増減ボタンまたは数値入力）
- 現在の在庫情報と指定人数をもとに、作れるレシピを一覧表示する
- 提案ロジックは以下の通り：
  - 各レシピの食材には「必須」または「任意」のフラグを持つ
  - **必須食材がすべて（指定人数分）在庫にある**、かつ**任意食材のうち1品以上が（指定人数分）在庫にある**場合にそのレシピを表示する
  - 任意食材が0品のレシピは、必須食材がすべて揃えば表示する
- レシピ詳細では「1人前」と「指定人数分」の両方の食材量を表示する

### 2.4 レシピ実行機能
- レシピ詳細画面の「作る」ボタンを押すと、指定人数分の使用量を在庫から自動減算する
- 在庫量が0以下になった食材は在庫から削除する

### 2.5 管理画面機能
- 食材マスタの追加・編集・削除
- レシピマスタの追加・編集・削除
  - 使用食材・1人前あたりの使用量・必須/任意フラグを設定できる

---

## 3. 非機能要件

- レスポンシブデザイン対応（スマートフォンからも利用可能）
- 操作がシンプルで家族全員が使えるUI

## 4. 技術スタック

| 項目 | 採用技術 |
|------|----------|
| フロントエンド | React / TypeScript |
| バックエンド | Node.js / Express / TypeScript |
| データベース | PostgreSQL |
| 認証 | メール＋パスワード認証 |
| デプロイ | Vercel |

---

## 5. データモデル（概要）

### users
| カラム | 型 | 説明 |
|--------|----|------|
| id | UUID | 主キー |
| email | string | メールアドレス |
| password_hash | string | ハッシュ化パスワード |
| created_at | timestamp | 作成日時 |

### ingredients（食材マスタ）
| カラム | 型 | 説明 |
|--------|----|------|
| id | UUID | 主キー |
| name | string | 食材名 |
| unit | string | 単位（例：g, 個, 本） |

### inventory（在庫：世帯共有）
| カラム | 型 | 説明 |
|--------|----|------|
| id | UUID | 主キー |
| ingredient_id | UUID | 食材ID（FK） |
| quantity | number | 在庫量 |

### recipes（レシピマスタ）
| カラム | 型 | 説明 |
|--------|----|------|
| id | UUID | 主キー |
| name | string | レシピ名 |
| description | string | 説明 |

### recipe_ingredients（レシピ×食材）
| カラム | 型 | 説明 |
|--------|----|------|
| id | UUID | 主キー |
| recipe_id | UUID | レシピID（FK） |
| ingredient_id | UUID | 食材ID（FK） |
| quantity_per_serving | number | 1人前あたりの使用量 |
| is_required | boolean | 必須フラグ |

---

## 6. 画面構成（概要）

| 画面 | 説明 |
|------|------|
| ログイン / 登録画面 | メール・パスワードで認証 |
| 在庫一覧画面 | 現在の食材在庫を表示・更新 |
| レシピ提案画面 | 今日の人数を入力し、在庫から作れるレシピを一覧表示 |
| レシピ詳細画面 | 1人前・指定人数分の食材量を表示、「作る」ボタン |
| 管理画面（食材） | 食材マスタのCRUD |
| 管理画面（レシピ） | レシピマスタのCRUD（1人前あたりの量・必須フラグ設定） |

---

## 7. 将来的な拡張（スコープ外）
